INTRODUCTION:

Currently, airline agents reaccommodate passengers to different planes via a blue screen command line, which is quite inefficient and challenging, as there are specific commands that each user must know in order to successfully operate it. As such, the idea of creating an admin user interface was introduced, and the Airline Solutions DX UX team was asked to help design the interface. I was tasked with designing a Seat Reaccommodation interface.

Redesign

Jane receives an alert saying that the plane assigned for Flight WWA101, Dallas to San Diego, is having issues and therefore Flight WWA101 needs to be assigned a new plane. As stated before, the current way to do this would be for Jane to go into the blue screen command line and enter codes to change this. However, with this user interface, Jane can easily do this.

Jane first would see the current seating arrangement for the plane. There are indicators for the different classes (in this mock, you only see blue). But she needs to choose a new plane and needs to make sure she chooses the one that produces the least amount of seat changes. In this example, she chooses the first plane that appears and wants to see the results.

In the second screen, she now is seeing the results of if she switched to MMB202. In this example, the first passenger listed (J. Burns) has switched seats.
